Questions

  1. The server in a client-server architecture saves and serves the data to the client upon a valid request.
  2. The client makes requests to a server, and receives (and often "visualizes") data.
  3. An IP address is a unique address used in requests. IP addresses are used to identify your device (phone, router, etc.) but also to geolocate you. IP addresses look like xxx.xxx.xxx.xxx where each "xxx" can be anywhere within one octet (0-255).
  4. A DNS server is essentially a phonebook, but instead of mapping phone numbers to names, it maps IP addresses of websites to a readable domain name like "google.com".
  5. This I was unsure about, so I researched it. According to scaler.com the Internet uses TCP/IP, which essentially is what most people think of as "the internet" (IP "navigates" and TCP "sends", making sure not data is missing), while WWW uses HTTP(S), which are the websites. So "Internet" is what I would call internet, whilst WWW can be seen as a set of all the websites featured.
  6. HTML is hypertext markup language. That means that it is a markup language, which means that it controls the form/structure of its layout (like .md), whilst also being able to create hypertext, which is text mixed with images, videos, sound, etc., but most importantly links to other HTML-files.
  7. In short, we include the CSS files inside the head and not the body because the head is loaded before the body. The body consists of the "physical" content on the website, aka. the images, the text, etc. The head features metadata and other "hidden" data the browser needs to correctly display the site, like CSS.
  8. In-line CSS override rules in a CSS file because the in-line CSS' style applies to this specific element, which takes priority over a CSS file's more general rules.
